# 买一送一商家后台 — 交互与视觉设计说明

> 依据 `frontend-design` skill 制定。本原型为**忠实复刻型企业后台**，不做营销化重设计，在截图基础上增强信息层级与操作反馈。

## 1. 设计定位

| 维度 | 决策 |
|------|------|
| 用途 | 餐饮 SaaS 商家运营人员配置/管理「买一送一」促销活动 |
| 受众 | B 端运营、店长，熟悉表格+表单+向导式配置 |
| 风格基调 | **Industrial / Utilitarian** — 高密度信息、功能优先、克制装饰 |
| 差异化记忆点 | **左侧实时手机预览**：表单变更即时反映 C 端商品标签与活动说明 |

## 2. 视觉系统

### 色彩（CSS Variables）

```
--brand-primary: #00B578      /* 主操作、步骤条激活、链接 */
--brand-primary-hover: #00A06A
--brand-primary-light: #E8F8F1 /* 提示条、选中背景 */
--bg-page: #F0F2F5            /* 页面底色 */
--bg-card: #FFFFFF
--border: #E8E8E8
--text-primary: #262626
--text-secondary: #8C8C8C
--text-link: #1890FF
--danger: #FF4D4F
--warning-bg: #FFF7E6
```

### 字体

- 中文：`"PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif`
- 数字/ID：等宽感通过 `font-variant-numeric: tabular-nums` 对齐表格列
- 避免 Inter/Roboto 等通用西文字体作为主字体

### 布局密度

- 侧边栏宽 200px，顶栏高 48px
- 表单 label 宽 120px，右对齐
- 表格行高 48px，紧凑但可点击
- 底部固定操作栏高 56px，带顶部分割线

### 动效（克制）

- 步骤切换：内容区 `opacity + translateY(8px)` 200ms
- 弹窗：遮罩 fade + 面板 scale(0.98→1) 180ms
- Toast 通知：自顶滑入 3s 自动消失
- 不做大面积渐变、粒子、花哨动效

## 3. 信息架构

```
营销管理
└── 促销活动
    └── 买一送一
        ├── 活动列表（默认页）
        └── 活动配置向导
            ├── Step1 活动基础规则
            ├── Step2 选择参与商品
            ├── Step3 选择参与门店
            └── Step4 预览并发布
```

## 4. 页面与交互规格

### 4.1 活动列表

**筛选区**

| 字段 | 控件 | 行为 |
|------|------|------|
| 活动名称/ID | 文本输入 | 模糊匹配 name 或 id |
| 活动状态 | 下拉 | 全部/草稿/待发布/未开始/进行中/已结束/已作废 |
| 活动时间 | 日期范围 | 与活动 start/end 有交集即命中 |
| 参与门店 | 下拉 | 全部/指定门店名 |

**状态 Tab**：与下拉联动，点击 Tab 覆盖状态下拉值。

**列表列**：活动ID、活动名称、活动商品、参与门店、活动时间、投放渠道、适用人群、状态、创建者、操作。

**行内操作（按状态显隐）**

| 状态 | 查看 | 编辑 | 复制 | 发布 | 作废 |
|------|:----:|:----:|:----:|:----:|:----:|
| 草稿 | ✓ | ✓ | ✓ | ✓ | — |
| 待发布 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 未开始 | ✓ | ✓ | ✓ | — | ✓ |
| 进行中 | ✓ | ✓ | ✓ | — | ✓ |
| 已结束 | ✓ | — | ✓ | — | — |
| 已作废 | ✓ | — | ✓ | — | — |

- **复制**：生成新活动，名称加「(副本)」，状态=草稿，id 重新生成
- **发布**：草稿/待发布 → 根据当前时间与活动时间计算为「未开始」或「进行中」
- **作废**：二次确认弹窗 → 状态改为已作废

### 4.2 配置向导 — Step1 活动基础规则

**适用渠道**

- 参与渠道：微信小程序 / 支付宝小程序 / 抖音小程序 / 其他（多选图标按钮）
- 参与订单类型：堂食 / 外卖 / 自提（多选）

**基础信息**

- 活动名称*、活动标签、活动说明、活动时间*（日期时间范围）
- 活动周期：每天 / 每周 / 每月
- 活动时段：全天 / 指定时段
- 活动优先级（数字）

**赠送规则**

- 规则类型：买A送A / 买A送B
- 买满数量*、赠送数量*（默认 1+1）
- 每单最多优惠次数
- 优惠叠加：同享商品级优惠、同享整单优惠、同享会员价、同享优惠券（是/否）
- 叠加计算规则：最优计算 / 顺序计算

**左侧手机预览**：展示活动标签、示例商品「买一送一」角标、活动名称与简短说明。

**底部按钮**：取消 | 保存草稿 | 下一步

### 4.3 Step2 选择参与商品

- 范围：全部商品 / 部分商品
- 部分商品时：分类筛选 + 「添加商品」弹窗
- 弹窗：左分类树 | 中商品表格（多选）| 右已选列表
- 已选表格列：商品名称、规格、类目、编码、条码、销售价、操作(移除)
- 校验：选「部分商品」时至少 1 个商品

### 4.4 Step3 选择参与门店

- 范围：全部门店 / 部分门店参与 / 部分门店不参与
- 部分时：三栏选择器（区域树 | 门店列表 | 已选门店）
- 校验：部分参与/不参与时至少 1 家门店

### 4.5 Step4 预览并发布

- 只读汇总：适用范围、基础信息、活动规则、参与商品表、参与门店表
- 底部：取消 | 上一步 | 保存草稿 | 发布

### 4.6 表单校验（阻断下一步）

- 活动名称必填
- 活动时间必填且结束 > 开始
- 买满/赠送数量 ≥ 1
- 商品/门店按范围规则校验

## 5. 状态流转

```
列表 --新建--> 向导(草稿可存)
向导 --保存草稿--> 列表[草稿]
向导 --发布--> 列表[未开始|进行中]
列表 --编辑--> 向导(回填)
列表 --复制--> 向导(副本草稿)
列表 --作废--> 列表[已作废]
```

## 6. 技术实现约束

- Vite + React 18 + TypeScript，无后端，localStorage 持久化活动数据
- 单页应用，路由用内部 state 切换（列表 / 向导 / 查看）
- 组件拆分：Layout、List、Wizard、ProductPicker、StorePicker、MobilePreview、Toast
